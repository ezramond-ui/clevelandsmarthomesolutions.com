# Cleveland Smart Home Solutions

> **Smart Living, Done Simply.** — A premium, fast, fully SEO-optimized
> marketing website for a smart home installation business serving
> Northeast Ohio.

Built as a lightweight static site generated by a small Node script (no
heavy framework), with serverless form handlers that email the owner.

---

## Quick start

```bash
npm install          # install dependencies (nodemailer, for the forms)
npm run build        # generate the site into /dist
npm run serve        # preview at http://localhost:3000
# or:
npm run dev          # build + serve in one step
```

The deployable site is the generated **`/dist`** folder.

## ✏️ Placeholders to update before launch

Edit **`src/data/site.js`** — everything marked `PLACEHOLDER`:

- **Phone** — `phone` and `phoneHref`
- **Email** — `email` (public) and `ownerEmail` (where form submissions go)
- **Address / ZIP** and map coordinates
- **`url`** — your real production domain (used for canonical tags, sitemap, schema)
- **Social links** (optional)

**Logo:** A placeholder mark (`CSH`) sits in the header. Drop your real logo
into `public/images/` (e.g. `logo.svg`) and swap the `.brand-logo` block in
`src/templates/layout.js`. **Founder photo:** placeholder lives on the About
page (`src/templates/pages/about.js`).

Run `npm run build` after any change.

## 📨 Making the forms send email

Both the **contact form** and the **review form** POST to serverless
functions in `/api`. They email submissions to the owner. **Reviews are
never published automatically** — they're emailed for approval; to publish
one, add it to `src/data/testimonials.js` and rebuild.

Set these environment variables (see `.env.example`):

```
SMTP_HOST, SMTP_PORT, SMTP_USER, SMTP_PASS   # your mail provider
MAIL_FROM   # optional; defaults to SMTP_USER
MAIL_TO     # owner inbox that receives submissions
```

Until SMTP is configured, the forms degrade gracefully: the visitor is
shown a friendly message asking them to call/email directly (no errors, no
data loss).

> Anti-spam: both forms include a hidden honeypot field and server-side
> validation.

## 🚀 Deploying

### Vercel (recommended — serverless forms work out of the box)
1. Push this repo to GitHub.
2. Import it in Vercel. `vercel.json` already sets the build command
   (`npm run build`), output directory (`dist`), and clean URLs.
3. Add the SMTP environment variables in **Project → Settings → Environment Variables**.
4. Deploy. The `/api/contact` and `/api/review` functions are detected automatically.

### Any static host (Netlify, Cloudflare Pages, S3, etc.)
- Build command: `npm run build` · Publish directory: `dist`
- For the email forms you'll need a serverless/function equivalent, or point
  the form `action`s at a form service. The handler logic lives in `/api`.

## 🗂 Project structure

```
build.js                 # static site generator (run via npm run build)
serve.js                 # local preview server + API routing
vercel.json              # host config (build, clean URLs, caching, headers)
.env.example             # SMTP config template
api/
  contact.js             # POST /api/contact  → emails quote requests
  review.js              # POST /api/review   → emails reviews for approval
  _lib/mailer.js         # shared nodemailer + validation helpers
public/                  # static assets copied verbatim into /dist
  css/styles.css         # the full design system
  js/main.js             # nav, AJAX forms, area filter
  images/                # favicon + social share image (SVG)
src/
  data/
    site.js              # ⭐ company info + PLACEHOLDERS
    services.js          # the 3 tiers + key services
    locations.js         # all 57 locations (drives the landing pages)
    testimonials.js      # approved/published reviews
  templates/
    layout.js            # HTML shell: SEO head, header, footer
    components.js        # reusable sections, icons, JSON-LD builders
    pages/               # one module per page type
dist/                    # generated output (the website) — git-ignored
```

## Adding or editing a location

Add a row to `src/data/locations.js` (name, county, type, nearby,
character, homes) and rebuild — a complete, unique, SEO-optimized landing
page is generated automatically.
